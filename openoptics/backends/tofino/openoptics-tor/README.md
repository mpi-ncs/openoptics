# Openoptics ToR

Optical data center network end-node running on Intel Tofino2 programmable switches. Time-flow table abstraction, time-based forwarding, and infrastructure services, e.g. queue depth estimation.

## Quick Start

### Automated deployment (recommended)

The OpenOptics control machine handles everything — P4 compilation, binary build, table population, and server IP setup:

```python
from openoptics import Toolbox, OpticalTopo, OpticalRouting

net = Toolbox.BaseNetwork(
    name="my_test",
    backend="Tofino",
    nb_node=4, nb_link=1,
    time_slice_duration_us=50,
    arch="direct",
    config_file="path/to/config.toml",
)

circuits = OpticalTopo.opera(nb_node=4, nb_link=1, guardband=True)
net.deploy_topo(circuits)
paths = OpticalRouting.routing_direct(net.get_topo())
net.deploy_routing(paths, routing_mode="Per-hop")
net.start()  # Opens CLI: h0 ping h1, server_check, etc.
```

The deployment pipeline:
1. Uploads P4 sources and config JSON to each switch via SSH
2. Compiles P4 with `-DSLICE_NUM=<nb_time_slices>` (from topology)
3. Builds and launches the C++ binary (`run.sh` reads all params from `openoptics_config.json`)
4. Runs `setup_tor.py` / `setup_ocs.py` via bfshell to populate tables
5. Connects to servers, brings up NICs, assigns data-plane IPs

### Manual build on the switch

If you need to build manually on the switch (e.g., for debugging):

```bash
# 1. Set up SDE environment
source /home/p4/bf-sde-9.12.0/set_sde.bash
export SDE_INSTALL=/home/p4/bf-sde-9.12.0/install

# 2. Compile P4 (SLICE_NUM is required — no default)
$SDE/p4_build.sh p4src/openoptics_tor.p4 --with-tofino2 -D SLICE_NUM=8

# 3. Ensure openoptics_config.json exists (normally uploaded by backend.py)
# It must contain: TOR_NUM, PORT_NUM, NB_TIME_SLICES, SLICE_DURATION, tors, etc.

# 4. Build and run the C++ binary
# run.sh reads all parameters from openoptics_config.json — no CLI args needed
./run.sh

# 5. In a separate terminal, run the setup script via bfshell
export SETUP_SCRIPT=$(pwd)/setup_tor.py
$SDE_INSTALL/bin/bfshell -b setup_tor.py
```

### Configuration

All runtime configuration flows through `openoptics_config.json`, generated by `backend.py` from `config.toml`. Key fields:

| JSON field | Source | Used by |
|---|---|---|
| `TOR_NUM` | `nb_node` in Python | C++ binary (argv via run.sh), setup_util.py |
| `PORT_NUM` | `nb_link` in Python | C++ binary (argv via run.sh), setup_util.py |
| `NB_TIME_SLICES` | Topology generation | C++ binary (argv via run.sh), setup_util.py, P4 compile flag |
| `SLICE_DURATION` | `time_slice_duration_us` | C++ binary (argv via run.sh), setup_tor.py |
| `ARCH` | `arch` kwarg | setup_tor.py (VLB tables) |
| `tors` | `config.toml` logical_tor entries | setup_tor.py (port/MAC/IP config per ToR) |
| `physical_switches` | `config.toml` physical_switch entries | setup_tor.py (hostname → tor_ids) |
| `ocs_fp_layout` | Derived from `tor_ocs_port_pairs` | setup_ocs.py (OCS cage mapping) |
| `bandwidth` | `config.toml` [bandwidth] | setup_tor.py, setup_ocs.py (port speed/FEC) |

### P4 compile-time constants

These must be set via `-D` flags to `p4_build.sh`:

| Constant | Flag | Description |
|---|---|---|
| `SLICE_NUM` | `-D SLICE_NUM=8` | **Required.** Number of time slices in the schedule. Must match `NB_TIME_SLICES`. |
| `PORT_NUM` | `-D PORT_NUM=4` | OCS uplink ports per ToR. Default: 4 (in `headers.p4`). |

`deploy.py` passes these automatically. For manual builds, you must provide them.

## Architecture

Three-layer stack:

```
┌─────────────────────────────────────────────┐
│  Python Configuration  (setup_tor.py)       │  BFRt table population at boot
├─────────────────────────────────────────────┤
│  C++ Control Plane     (openoptics_tor.cpp) │  BF switchd, pktgen, AFC setup
├─────────────────────────────────────────────┤
│  P4 Data Plane         (p4src/)             │  Tofino2 packet pipeline
└─────────────────────────────────────────────┘
```

### Data flow

```
config.toml (control machine)
  → backend.py generates openoptics_config.json + tor_entries_torN.json + schedule.txt
    → deploy.py uploads via SCP, compiles P4 with -DSLICE_NUM, builds C++ binary
      → run.sh reads JSON, passes args to C++ binary (duration, nb_time_slices, tor_num, port_num)
        → C++ binary starts bf_switchd, configures pktgen/AFC
          → deploy.py runs setup_tor.py via bfshell (reads JSON for table entries)
```

## Data Plane (P4)

**Source**: `p4src/`

### Ingress Pipeline

Two parallel sub-pipelines process each packet:

1. **QueueManager** (`queue.p4`) — handles queuing control packets:
   - `ETHERTYPE_ROTATION (0x3001)`: Slice transition signal. Updates the current slice register and broadcasts AFC pause/resume messages.
   - `ETHERTYPE_UPDATEQD (0x3002)`: Queue drain tick. Subtracts `qdiff` bytes from per-port queue depth registers every 50 ns.
   - Pktgen app_id 7: One-shot kickoff that triggers initial AFC pause on startup.

2. **RoutingDecision** (`routing.p4`) — handles normal traffic:
   - Looks up `time_flow_table` keyed on `(cur_slice, dst_group)` to get the egress port, active send-slice, and next-hop ToR.
   - Runs per-port admission control: checks estimated queue depth against threshold; reroute if full.
   - Updates the estimated queue depth register on admission.

Egress is bypassed; all forwarding decisions are made in ingress.

### Key P4 Tables

| Table | Key | Purpose |
|---|---|---|
| `time_flow_table` | `(cur_slice, dst_group)` | Primary routing; returns port, send_slice, next_tor, alternates |
| `set_afc_tb` | `(app_id, slice_id, packet_id)` | Emits 32-bit AFC pause/resume control words |
| `tb_compute_p{i}_active_rank` | `cur_slice` | Maps current slice → active queue rank for port i |
| `tb_compute_p{i}_rank` | `next_hop MAC` | Maps destination → queue rank for port i |
| `tb_set_egress_port` | logical port | Maps logical port ID → physical dev_port |
| `tb_set_new_slice_for_p{i}` | `cur_slice` | Resets lossless threshold when port-queue transitions to active |
| `tb_adjust_optics_frame_size` | EtherType | Adds optics encapsulation overhead to frame size field |
| `tb_check_to_server` | dst MAC | Redirects locally-destined packets to server port (0xff) |

### Lossless Flow Control (AFC)

Advanced Flow Control to pause/resume queues to support time-based routing:

- All OCS-facing port-queues operate in `XOFF` (pause-on-demand) mode.
- When a circuit becomes **inactive**, the switch sends an AFC **pause** message to that port-queue, stopping packet sending out.
- When the circuit becomes **active** again, an AFC **resume** message reactivates the queue.
- AFC control messages are pre-computed per `(slice, port, queue)` and stored in `set_afc_tb`; the data plane emits them during slice transitions.

### Queue Depth Estimation

Since Tofino hardware queue depth (or queuing delay) is not directly accessible at the ingress with required accuracy, we use a register to keep track of the estimated queue depth in nanosecond-level accuracy:

- Each port maintains two registers: `p{i}_qdepth_reg_table` (one entry per queue).
- On packet admission, the register is incremented by the frame size.
- A pktgen app fires with a fixed interval (every **50 ns** by default), emitting `ETHERTYPE_UPDATEQD` packets that drain registers by `qdiff = 50 ns × 100 Gbps / 8 ≈ 625 bytes/interval`.
- If queue depth ≥ `max_lossless_qdepth`, the packet is rerouted or dropped at ingress.

## Control Plane (C++)

**Source**: `openoptics_tor.cpp`

### Command-line interface

```
./openoptics_tor <duration> <nb_time_slices> <tor_num> <port_num> [--daemon]
```

All arguments are passed by `run.sh` which reads them from `openoptics_config.json`.

| Argument | Example | Description |
|---|---|---|
| `duration` | `50us` | Slice duration |
| `nb_time_slices` | `8` | Total time slices in schedule |
| `tor_num` | `4` | Number of ToR nodes |
| `port_num` | `1` | OCS uplink ports per ToR |
| `--daemon` | | Stay alive in headless mode (set by deploy.py) |

### Initialization Sequence

1. **`init_bf_switchd()`** — loads the P4 program via BF switchd from `$SDE_INSTALL/share/p4/targets/tofino2/openoptics_tor.conf`.
2. **`init_afc()`** — enables AFC XOFF mode on all 4 pipes for all OCS-facing port-queues.
3. **pktgen setup** — configures queue drain (50ns periodic) and AFC kickoff (one-shot) apps.
4. **`--daemon` mode** — enters infinite sleep for deploy.py to run `setup_tor.py` via bfshell. Without `--daemon`, drops into interactive bfshell.

### Pktgen Applications

| App | ID | Mode | Period | Purpose |
|---|---|---|---|---|
| Queue drain | 5 | Periodic | 50 ns | Fires `ETHERTYPE_UPDATEQD` to drain virtual queue registers |
| AFC kickoff | 1 | One-shot | — | Sends initial AFC pause for all queues before first time slice |
| Rotation | 3 | Periodic | `slice_duration` | Optional: drives AFC cycle (emulated OCS sends this to ToRs for synchronization) |

## Python Configuration

**Source**: `setup_tor.py`, utility functions in `setup_util.py`

All configuration is read from `openoptics_config.json` (no environment variables, no hardcoded values). `setup_tor()` runs once per pipe at boot and populates all P4 tables for that ToR:

1. Reads per-ToR hardware config (port cages, MACs, IPs) from JSON
2. Resolves front-panel cage/lane to dev_port at runtime via `bfrt.port.port_hdl_info`
3. Configures physical ports with speed/FEC from config (with per-ToR overrides)
4. Populates queue drain parameters and AFC pause/resume tables from schedule
5. Loads routing entries from `tor_entries_torN.json` into `time_flow_table`
6. Maps logical ports to physical dev_ports in `tb_set_egress_port`

### OCS Schedule

`schedule.txt` — circuit schedule matrix generated by `backend.py` from the deployed topology:
- Rows = time slices
- Columns = port 0..nb_link-1 for each of ToR 0..nb_node-1
- Value = destination ToR index, or `-1` if no circuit is active (guardband)

The Tofino backend **requires** guardband time slices in the topology (use `guardband=True` in topology generation). Without guardbands, `setup_util.py`'s AFC functions will fail.

## Directory Structure

```
openoptics-tor/
├── p4src/
│   ├── openoptics_tor.p4     # Top-level ingress/egress pipeline
│   ├── queue.p4              # Slice transition and AFC signal handling
│   ├── routing.p4            # Normal traffic routing with admission control
│   └── common/
│       ├── headers.p4        # Packet header definitions and compile-time constants
│       ├── util.p4           # Tofino parser boilerplate
│       └── reg.p4            # Per-port queue depth register declarations
├── hoho_table/               # Legacy pre-computed HoHo routing entries (reference only)
├── opera_table/              # Legacy pre-computed OPERA routing entries (reference only)
├── schedule.txt              # OCS circuit schedule matrix (generated by backend.py)
├── openoptics_tor.cpp        # C++ control plane source
├── setup_tor.py              # BFRt Python table configuration script
├── setup_util.py             # Schedule parsing and slice/rank utility functions
├── Makefile                  # Build rules for the C++ binary
└── run.sh                    # Entry point: reads JSON config, builds C++, launches binary
```
