# Emulated OCS

Optical Circuit Switch (OCS) emulation running on Intel Tofino/Tofino2. A single physical switch emulates the OCS role in the optical data center network, working in conjunction with [openoptics-tor](../openoptics-tor/).

## Role in the System

```
 ToR 0 в”Җв”Җв”җ
 ToR 1 в”Җв”Җв”Ө
 ToR 2 в”Җв”Җв”Ө  emulated-ocs  (this module)
 ToR 3 в”Җв”Җв”Ө    - Forward optics packets to the next-hop ToR
 ToR 4 в”Җв”Җв”Ө    - Broadcast rotation (pause/resume) signals to all ToRs
 ToR 5 в”Җв”Җв”Ө      for slice-transition synchronization
 ToR 6 в”Җв”Җв”Ө
 ToR 7 в”Җв”Җв”ҳ
```

The OCS does not schedule traffic. It provides two services:
1. **Optical forwarding** вҖ” `ETHERTYPE_OPTICS` packets are forwarded to the next-hop ToR based on the `next_tor` field in the optics L2 header.
2. **Circuit signaling** вҖ” periodic `ETHERTYPE_ROTATION` packets are multicast to all connected ToRs to trigger slice transitions.

## Requirements

- Intel Barefoot SDE 9.12.0
- `SDE` and `SDE_INSTALL` environment variables set

## Usage

Invoked automatically by `deploy.py` from the control machine; no manual CLI arguments.

```bash
./run.sh
```

`run.sh` reads `SLICE_DURATION` and `NB_TIME_SLICES` from `openoptics_config.json` (generated by `backend.py`), builds the binary, and launches it with `sudo -E` to preserve SDE variables. Valid `SLICE_DURATION` values: `1`, `2`, `10`, `25`, `50`, `100`, `500` (Вөs).

`deploy.py` sets `OPENOPTICS_DAEMON=1` so the binary stays headless and accepts BFRt gRPC on port `50052` for remote setup.

## Data Plane (P4)

**Source**: `p4src/ocs.p4`

### Ingress Parser

Detects pktgen packets by `app_id` lookahead before parsing Ethernet:

| `app_id` | Constant | Treatment |
|---|---|---|
| `3` | `PKTGEN_APP_ID_ROTATION` | Strip pktgen header, parse Ethernet |
| `7` | `PKTGEN_APP_ID_KICKOFF` | Strip pktgen header, parse Ethernet |
| other | вҖ” | Parse Ethernet directly |

After Ethernet, the parser selects on EtherType:

| EtherType | Constant | Action |
|---|---|---|
| `0x3001` | `ETHERTYPE_ROTATION` | Accept (no further parsing) |
| `0x3000` | `ETHERTYPE_OPTICS` | Parse `optics_l2` header |
| `0x3003` | `ETHERTYPE_OPTICS_SR` | Source-routed optics frame; accept |
| `0x0800` | `ETHERTYPE_IPV4` | Accept |
| `0x86dd` | `ETHERTYPE_IPV6` | Accept |

### Ingress Control

| Packet type | Forwarding action |
|---|---|
| `ETHERTYPE_ROTATION` | Multicast to all ToR-facing ports (`MCAST_GRP_ID = 1`) |
| `ETHERTYPE_OPTICS` | Unicast via `ocs` table keyed on `optics_l2.next_tor` |
| `ETHERTYPE_OPTICS_SR` | Unicast via `ocs` table keyed on `ethernet.dst_addr` (next-hop MAC pre-baked at source) |
| IPv4 / IPv6 | Unicast via `ocs` table keyed on `ethernet.dst_addr` |

The `ocs` table (128 entries) maps a MAC address to a physical egress port. Egress pipeline is bypassed.

### Key Constants

| Constant | Value | Description |
|---|---|---|
| `ETHERTYPE_OPTICS` | 0x3000 | Optics-encapsulated data frame |
| `ETHERTYPE_ROTATION` | 0x3001 | Slice transition signal (multicast) |
| `ETHERTYPE_OPTICS_SR` | 0x3003 | Source-routed optics frame |
| `PKTGEN_APP_ID_KICKOFF` | 7 | Pktgen kickoff app |
| `PKTGEN_APP_ID_ROTATION` | 3 | Pktgen rotation timer app |
| `MCAST_GRP_ID` | 1 | Multicast group for rotation broadcast |

The schedule length (`SLICE_NUM`) is passed in at runtime via the second CLI argument of the `ocs` binary; no compile-time constant is required.

## Control Plane (C++)

**Source**: `ocs.cpp`

### CLI

```
./ocs <duration> <nb_time_slices> [--daemon]
```

| Argument | Example | Description |
|---|---|---|
| `duration` | `50us` | Slice duration |
| `nb_time_slices` | `8` | Total time slices in schedule |
| `--daemon` | | Stay alive in headless mode (set by `deploy.py` via `OPENOPTICS_DAEMON=1` in `run.sh`) |

### Initialization Sequence

1. Parse positional args; convert duration to nanoseconds (`slice_duration = duration Г— 1000`).
2. **`init_bf_switchd("ocs")`** вҖ” loads P4 program from `$SDE_INSTALL/share/p4/targets/tofino2/ocs.conf`.
3. Sleep 10 s for switchd to settle.
4. **`set_rotation_pktgen()`** вҖ” starts periodic rotation signal generation (batch count derived from `slice_num`).
5. **Daemon mode**: infinite sleep; table population happens out-of-band via `deploy.py` over the BFRt gRPC channel. Without `--daemon`, drops into interactive `bfshell`.

### Pktgen Application

| App | ID | Mode | Period | Purpose |
|---|---|---|---|---|
| Rotation | `PKTGEN_APP_ID_ROTATION` (3) | Periodic | `slice_duration` ns | Generates `ETHERTYPE_ROTATION` packets broadcast to all ToRs |

The rotation app fires with `batch_count = slice_num / 2 - 1` (one batch per pair of slices, derived from the runtime `nb_time_slices`), one packet per batch at `ipg = slice_duration`.

## Python Configuration

**Source**: `setup_ocs.py`

Reads `TOR_NUM`, `NB_TIME_SLICES`, `ocs_fp_layout`, and `bandwidth` from `openoptics_config.json` (uploaded by `deploy.py`) and configures the switch:

1. **`clear_all()`** вҖ” clears all match, register, selector, and action-profile tables.
2. **Port configuration** вҖ” enables the front-panel cages listed in `ocs_fp_layout` at the configured speed/FEC.
3. **`ocs` table** вҖ” populates `(next_tor вҶ’ egress dev_port)` entries from `ocs_fp_layout`.
4. **Multicast group** вҖ” defines the `MCAST_GRP_ID = 1` group covering all ToR-facing uplink dev_ports for rotation signal broadcast.

All port/MAC/cage mappings are config-driven; nothing is hardcoded in `setup_ocs.py`. `ocs_fp_layout` is emitted by `backend.py` as a list of `[tor_id, port_idx, "cage/lane"]` triples derived from each logical ToR's `tor_ocs_port_pairs` in `config.toml`:

```toml
# config_4tor.toml (excerpt)
[[physical_switch.logical_tor]]
tor_id             = 0
tor_ocs_port_pairs = [["7/0", "7/0"]]   # (tor_fp, ocs_fp) per uplink
```
